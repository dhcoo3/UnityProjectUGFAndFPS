## 系统概述

这是一套面向**银河恶魔城（Metroidvania）**类型的横版动作系统，基于**状态机模式**实现，核心设计围绕**平台跳跃 + 探索 + 战斗**。

### 核心特性
- **14种动作状态**：待机、移动、跳跃、下落、攻击、空中下砸、技能、受击、下蹲、翻滚、冲刺、死亡、贴墙下滑、蹬墙跳
- **攻击系统**：前摇+判定两阶段、2段连击
- **平台跳跃**：可控跳跃高度、二段跳（解锁）、贴墙下滑、蹬墙跳
- **能力解锁**：冲刺、二段跳、空中下砸等能力需探索获取
- **输入缓冲**：1个输入缓冲，保证操作响应
- **翻滚无敌帧**：0.25秒无敌时间

### 优先级系统（4档简化）
```
死亡(3) > 受击(2) > 动作类(1) > 普通(0)

优先级0：待机、移动、下蹲
优先级1：跳跃、下落、贴墙下滑、蹬墙跳、攻击、空中下砸、技能、冲刺、翻滚
优先级2：受击
优先级3：死亡
```

## 核心模块说明

### 1. ActionConfig.cs - 动作配置
定义每个动作的属性（银河恶魔城简化版）：
```csharp
public class ActionConfig
{
    public ActionType ActionType;      // 动作类型
    public int Priority;               // 优先级（0-3）
    public bool CanBeInterrupted;      // 是否可被打断
    public float Duration;             // 持续时间
    public float CooldownTime;         // 冷却时间
    public bool AllowInputBuffer;      // 是否允许输入缓冲
    public float CancelableTime;       // 可取消时间点
    public float AttackDelay;          // 攻击前摇时间
}
```

### 2. ActionStateMachine.cs - 状态机
核心状态管理器，负责：
- 状态注册与切换
- 冷却时间管理（缓存key列表，零GC）
- 输入缓冲队列（1个）
- 状态转换规则验证

**关键方法：**
```csharp
// 注册状态
stateMachine.RegisterState(state, config);

// 切换状态
stateMachine.ChangeState(ActionType.Attack);

// 获取状态实例
var jumpState = stateMachine.GetState<JumpState>(ActionType.Jump);

// 获取冷却时间
float cooldown = stateMachine.GetCooldownRemaining(ActionType.Skill);
```

### 3. ActionInputManager.cs - 输入管理
处理玩家输入并转换为动作命令：
```csharp
inputManager.OnMoveInput(horizontal);     // 移动
inputManager.OnJumpInput();               // 跳跃（自动处理二段跳/蹬墙跳）
inputManager.OnJumpRelease();             // 控制跳跃高度
inputManager.OnAttackInput();             // 攻击
inputManager.OnAirAttackInput();          // 空中下砸
inputManager.OnSkillInput(skillId);       // 技能
inputManager.OnDashInput(direction);      // 冲刺
inputManager.OnDodgeInput();              // 翻滚
inputManager.OnCrouchInput(isPressed);    // 下蹲
inputManager.OnWallContact(isTouching);   // 贴墙检测（物理系统调用）
```

### 4. HitBoxSystem.cs - 判定盒系统
定义攻击判定数据（简化版）：
```csharp
public class AttackData
{
    public float Damage;              // 伤害值
    public Vector2 KnockbackForce;    // 击退力度
}
```

### 5. AnimationEventReceiver.cs - 动画事件接收器
接收Unity动画事件并转发给控制器：
```csharp
OnAttackStart()                       // 攻击开始
OnAttackHit()                         // 攻击判定帧
OnAttackEnd()                         // 攻击结束
OnFootstep()                          // 脚步声
OnSpawnEffect(string effectName)      // 生成特效
```

## 状态详解

### 1. 待机状态 (IdleState)
- **优先级**：0
- **特点**：可切换到任何状态，进入时清空输入缓冲

### 2. 移动状态 (RunState)
- **优先级**：0
- **特点**：带加速度/减速度，由输入控制
- **参数**：移动速度 5m/s，加速度 20m/s²，减速度 15m/s²

### 3. 跳跃状态 (JumpState)
- **优先级**：1
- **特点**：按键时长控制高度，支持二段跳（需解锁）
- **参数**：
  - 跳跃初速度：10 m/s
  - 短跳速度：5 m/s
  - 重力：20 m/s²
  - 空中控制：80%
- **可转换到**：Fall、WallSlide、Attack、AirAttack、Skill、Dodge、Hit、Dead

### 4. 下落状态 (FallState)
- **优先级**：1
- **特点**：从跳跃自动切换，支持贴墙转换
- **参数**：最大下落速度 15m/s，空中控制 80%
- **可转换到**：Idle(落地)、WallSlide、Jump(二段跳)、Attack、AirAttack、Dodge、Hit、Dead

### 5. 攻击状态 (AttackState)
- **优先级**：1
- **特点**：前摇+判定两阶段，2段连击
- **参数**：前摇 0.08秒，总时长 0.4秒，冷却 0.15秒
- **连击数据**：
  - 第1段：10伤害，轻击退
  - 第2段：18伤害，重击退
- **取消规则**：前摇仅受击/死亡可打断，判定后可被翻滚/跳跃/攻击/技能取消

### 6. 空中下砸 (AirAttackState)
- **优先级**：1
- **特点**：滞空0.15秒→高速下砸→落地冲击，需解锁
- **参数**：下砸速度 25m/s，伤害 20
- **可转换到**：Idle(落地)、Hit、Dead

### 7. 技能状态 (SkillState)
- **优先级**：1
- **特点**：霸体（仅受击/死亡可打断），支持技能ID
- **参数**：前摇 0.15秒，总时长 0.8秒，冷却 3秒

### 8. 受击状态 (HitState)
- **优先级**：2
- **特点**：击退衰减，仅死亡可打断
- **参数**：持续 0.25秒

### 9. 下蹲状态 (CrouchState)
- **优先级**：0
- **特点**：由输入控制，用于穿越低矮通道
- **可转换到**：Idle、Attack、Hit、Dead

### 10. 翻滚状态 (DodgeState)
- **优先级**：1
- **特点**：0.25秒无敌帧
- **参数**：持续 0.35秒，冷却 0.6秒，翻滚速度 12m/s

### 11. 冲刺状态 (DashState)
- **优先级**：1
- **特点**：快速水平位移，需解锁
- **参数**：持续 0.25秒，冷却 0.8秒，冲刺速度 18m/s

### 12. 贴墙下滑 (WallSlideState)
- **优先级**：1
- **特点**：降低下落速度，由物理检测触发
- **参数**：下滑速度 2m/s
- **可转换到**：WallJump、Jump、Fall、Idle(落地)、Attack、Hit、Dead

### 13. 蹬墙跳 (WallJumpState)
- **优先级**：1
- **特点**：反方向弹射，垂直+水平速度
- **参数**：垂直速度 10m/s，水平速度 8m/s
- **可转换到**：Fall、Attack、Hit、Dead

### 14. 死亡状态 (DeadState)
- **优先级**：3（最高）
- **特点**：不可切换到其他状态
- **持续时间**：2秒

## 能力解锁系统

银河恶魔城的核心玩法循环是**探索→获取新能力→解锁新区域**。

### 使用方式
```csharp
var controller = GetComponent<PlayerActionController>();

// 解锁冲刺
controller.UnlockAbility(ActionType.Dash);

// 解锁二段跳（特殊处理，自动设置MaxJumpCount=2）
controller.UnlockAbility(ActionType.Jump);

// 解锁空中下砸
controller.UnlockAbility(ActionType.AirAttack);

// 检查是否已解锁
bool hasDash = controller.IsAbilityUnlocked(ActionType.Dash);
```

### 默认解锁状态
| 能力 | 默认状态 | 说明 |
|------|---------|------|
| 移动/跳跃/攻击/翻滚 | 已解锁 | 基础能力，无需解锁 |
| 冲刺 (Dash) | 需解锁 | 快速水平位移 |
| 二段跳 (Jump) | 需解锁 | 空中再次跳跃 |
| 空中下砸 (AirAttack) | 需解锁 | 空中向下攻击 |

## 扩展开发

### 1. 添加新动作状态

**步骤1：在ActionType枚举中添加**
```csharp
public enum ActionType
{
    // ...现有状态
    Climb = 15,  // 攀爬
}
```

**步骤2：创建状态类**
```csharp
public class ClimbState : ActionStateBase
{
    public override void Enter()
    {
        base.Enter();
        // 进入逻辑
    }

    public override void Update(float deltaTime)
    {
        ElapsedTime += deltaTime;
        // 更新逻辑（不调用base.Update可避免Duration自动完成）
    }

    public override bool CanTransitionTo(ActionStateBase targetState)
    {
        ActionType target = targetState.Config.ActionType;
        return target == ActionType.Idle
            || target == ActionType.Jump
            || target == ActionType.Hit
            || target == ActionType.Dead;
    }
}
```

**步骤3：注册状态**
```csharp
_stateMachine.RegisterState(
    new ClimbState(),
    new ActionConfig(ActionType.Climb, priority: 1,
        canBeInterrupted: true, duration: 0f)
);
```

### 2. 自定义攻击数据

修改 `AttackState` 构造函数中的连击数据：
```csharp
_comboAttackData = new AttackData[]
{
    new AttackData(damage: 15f, knockbackForce: new Vector2(4f, 1f)),
    new AttackData(damage: 25f, knockbackForce: new Vector2(6f, 2f)),
};
```

### 3. 实现攻击判定检测

在 `AttackState.OnEnterActivePhase()` 中：
```csharp
private void OnEnterActivePhase()
{
    HitBoxData attackBox = CurrentAttackData.AttackBox;
    attackBox.IsActive = true;

    Collider2D[] hits = Physics2D.OverlapBoxAll(
        attackBox.GetWorldRect(transform.position, facingRight).center,
        attackBox.Size, 0f, enemyLayer
    );

    foreach (var hit in hits)
    {
        var enemy = hit.GetComponent<PlayerActionController>();
        enemy?.TakeDamage(new HitData
        {
            Damage = CurrentAttackData.Damage,
            KnockbackDirection = facingRight ? Vector2.right : Vector2.left,
            KnockbackForce = CurrentAttackData.KnockbackForce.x,
            Attacker = gameObject
        });
    }
}
```

## 常见问题

### Q1: 攻击没有判定？
检查：
1. 是否在动画中添加了 `OnAttackHit` 事件
2. 是否挂载了 `AnimationEventReceiver` 组件
3. 是否实现了攻击判定检测代码

### Q2: 跳跃高度无法控制？
确保调用了 `OnJumpRelease()`：
```csharp
if (Input.GetKeyUp(KeyCode.Space))
{
    _inputManager.OnJumpRelease();
}
```

### Q3: 翻滚没有无敌效果？
在受伤判定中已自动检查无敌状态：
```csharp
if (_stateMachine.CurrentState is DodgeState dodgeState && dodgeState.IsInvincible)
{
    return; // 免疫伤害
}
```

### Q4: 二段跳不生效？
需要先解锁：
```csharp
controller.UnlockAbility(ActionType.Jump);
```

### Q5: 冲刺/空中下砸无法使用？
这些是需要解锁的能力：
```csharp
controller.UnlockAbility(ActionType.Dash);       // 解锁冲刺
controller.UnlockAbility(ActionType.AirAttack);   // 解锁空中下砸
```

### Q6: 贴墙下滑不触发？
需要外部物理检测调用：
```csharp
// 在碰撞检测中
_inputManager.OnWallContact(isTouchingWall);
```

### Q7: 如何添加屏幕震动？
在空中下砸落地或攻击命中时触发：
```csharp
public void OnAnimationEvent_Custom(string eventName)
{
    if (eventName == "ScreenShake")
    {
        CameraShake.Shake(0.1f, 0.05f);
    }
}
```

## 状态转换总览

```
Idle     → Run, Jump, Crouch, Attack, Skill, Dash, Dodge, Hit, Dead
Run      → Idle, Jump, Crouch, Attack, Skill, Dash, Dodge, Hit, Dead
Jump     → Fall, WallSlide, Attack, AirAttack, Skill, Dodge, Hit, Dead
Fall     → Idle(落地), WallSlide, Jump(二段跳), Attack, AirAttack, Dodge, Hit, Dead
WallSlide→ WallJump, Jump, Fall, Idle(落地), Attack, Hit, Dead
WallJump → Fall, Attack, Hit, Dead
Crouch   → Idle, Attack, Hit, Dead
Attack   → Idle, Jump, Dodge, Attack(连击), Skill, Hit, Dead
AirAttack→ Idle(落地), Hit, Dead
Skill    → Idle, Hit, Dead
Dash     → Idle, Attack, Hit, Dead
Dodge    → Idle, Attack, Jump, Hit, Dead
Hit      → Idle, Dead
Dead     → (无)
```

## 性能优化说明

相比原格斗版本的优化：
- **冷却更新零GC**：使用缓存key列表替代每帧new List
- **删除连招字符串拼接**：无每帧string.Join和EndsWith开销
- **输入缓冲缩减**：3→1，减少队列操作
- **配置精简**：ActionConfig字段从11个减少到8个
