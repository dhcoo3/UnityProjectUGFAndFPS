# 自动开发自测模板

本文档用于统一自动开发完成后的自测输出格式，确保每次改动都能快速验收和回报。

## 1. 使用目标

每次自动开发完成后，至少回答四个问题：

- 改了什么
- 怎么验
- 验证结果是什么
- 还有什么风险

## 2. 自测前检查

在开始自测前，先确认：

- Unity 工程已打开
- MCP 已连接
- 相关配置已导表
- 目标场景或模块已准备
- 相关日志窗口可见

## 3. 通用自测步骤

### 3.1 基础信息

先记录：

- 任务名称
- 关联文档
- 修改文件
- 目标模块
- 预期结果

### 3.2 功能路径

按实际改动选择路径：

- 启动流程
- 登录流程
- 章节入口
- 进图流程
- 战斗流程
- 结算流程
- UI 打开关闭
- 配置导表

### 3.3 日志检查

重点检查：

- 是否有 `Error`
- 是否有 `NullReference`
- 是否有配置缺失日志
- 是否有重复注册日志
- 是否有结算重复触发日志

### 3.4 回归检查

至少确认：

- 主流程仍可进入
- 原有入口未失效
- 原有配置未被破坏
- 相关 UI 未残留状态

## 4. 分模块自测模板

### 4.1 逻辑 / 战斗

检查项：

- 是否能进入战斗
- 玩家是否可操作
- 子弹或技能是否生效
- 伤害是否正确结算
- 怪物死亡是否正常
- 刷怪是否按条件触发

### 4.2 地图 / 关卡

检查项：

- 场景是否正确加载
- 碰撞数据是否正常读取
- 出生点是否正确
- 平台或地形是否正常
- 终点或胜利条件是否可触发

### 4.3 UI

检查项：

- 面板能否打开
- 按钮是否可点击
- 重复打开是否正常
- 关闭后是否残留状态
- 数据刷新是否正确

### 4.4 配置

检查项：

- 导表是否成功
- 外键是否有效
- 必填字段是否完整
- 枚举是否合法
- 运行时是否可读

### 4.5 结算

检查项：

- 胜利结算是否出现
- 失败结算是否出现
- 返回主菜单是否正常
- 重新挑战是否正常
- 重复结算是否被阻止

## 5. 自测记录模板

建议按以下格式输出：

```text
任务：XXX
目标：XXX
修改文件：
- xxx.cs
- xxx.md

验证步骤：
1. ...
2. ...

验证结果：
- 通过 / 未通过

风险：
- ...
```

## 6. 问题回报模板

如果自测失败，必须按以下格式回报：

```text
问题：XXX
复现步骤：
1. ...
2. ...
期望结果：XXX
实际结果：XXX
影响范围：XXX
日志关键字：XXX
```

## 7. 最小验收标准

自动开发的每次输出至少满足：

- 说明改了哪些文件
- 说明做了什么
- 说明怎么验证
- 说明还有什么风险
