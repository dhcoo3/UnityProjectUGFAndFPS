# 结算与退出流程说明

本文档用于统一战斗结算、退出战斗、返回主菜单和重新挑战的流程。

## 1. 目标

确保以下情况都能稳定收口：

- Boss 或普通敌人死亡后进入胜利结算。
- 玩家死亡后进入失败结算。
- 结算后可返回主菜单。
- 结算后可重新挑战当前关卡。
- 退出战斗时不会残留角色、怪物、事件和 UI 状态。

## 2. 相关入口

优先对齐以下逻辑入口：

- `FightProxy.EnterFight`
- `FightProxy.EndFight`
- `FightProxy.ExitFight`
- `FightMainChapterController.EnterFight`
- `FightMainChapterController.ExitFight`
- `FightMainChapterController.InitFightFSM`
- `GameOverProcedure.OnEnter`
- `GameOverProcedure.OnLeave`
- `GameOverProcedure.ShowGameOverUIForm`
- `GameOverProcedure.BackHome`

## 3. 胜利流程

建议流程：

1. 战斗条件满足胜利。
2. `FightProxy.EndFight` 收到胜利信号。
3. 停止刷怪、停止输入、冻结战斗更新。
4. 关闭战斗中 UI 或切换到结算态 UI。
5. `GameOverProcedure` 展示胜利结果。
6. 玩家选择返回主菜单或重新挑战。

## 4. 失败流程

建议流程：

1. 玩家 HP 归零或满足失败条件。
2. `FightProxy.EndFight` 收到失败信号。
3. 停止战斗更新和战斗事件。
4. 清理场景内临时对象和战斗状态。
5. `GameOverProcedure` 展示失败结果。
6. 玩家选择返回主菜单或重新挑战。

## 5. 重新挑战

建议流程：

1. 关闭结算 UI。
2. 清理当前战斗实例。
3. 调用 `ExitFight` 或等价清理流程。
4. 重新调用 `EnterFight`。
5. 重新加载地图、单位、刷怪和 HUD。

## 6. 返回主菜单

建议流程：

1. 结束当前战斗上下文。
2. 释放战斗中临时对象。
3. 回到主菜单或章节入口。
4. 恢复入口 UI 状态。
5. 确认再次进入不会残留旧战斗数据。

## 7. 清理规则

退出或结算时必须清理：

- 怪物刷怪计时器
- 技能时间线
- 子弹与特效
- 受击和伤害事件
- 输入锁定状态
- HUD 战斗态
- 结算重复触发标记

## 8. 验收清单

- 胜利后能正常结算。
- 失败后能正常结算。
- 返回主菜单不报错。
- 重新挑战能重新进图。
- 连续进出关卡后不残留旧对象。
- 结算不会重复触发。
