# 配置校验与容错规范

本文档用于约束自动开发在面对配置缺失、字段错误、外键失效时的处理方式。

## 1. 目标

保证自动开发遵守统一原则：

- 配置有问题时先暴露问题，不要猜。
- 必须阻断的错误立即停下。
- 可以降级的错误明确降级，不要静默失败。
- 日志里必须能定位到表名、字段名、记录 ID。

## 2. 适用范围

本规范适用于以下表和数据：

- `MapData`
- `MainChapter`
- `PlatformDef`
- `MonsterSpawner`
- `Monster`
- `AIGroupDef`
- `AIActionDef`
- `SkillDef`
- `TimeLineDef`
- `BulletDef`
- `SkillBulletDef`
- `SkillAoeDef`
- `AoeDef`
- `BuffDef`
- `GameModule`
- `RedDot`
- `Localization`

## 3. 校验优先级

### 3.1 必须阻断

以下错误发现后应停止继续联调或自动补逻辑：

- 主键重复
- 必填字段为空
- 外键引用不存在
- 枚举值非法
- 配置表结构与导表代码不一致
- 关卡入口找不到 `MapId`
- 结算流程关键入口缺失

### 3.2 可降级

以下情况可降级，但必须记录明确日志：

- `Music` 为空
- 视觉特效资源缺失
- 文本本地化缺失
- 非关键 UI 入口缺失
- 次要表现资源缺失

### 3.3 允许默认值

以下情况可在明确默认值下继续：

- `PlatformDataId` 为空时，地图不含动态平台
- `AddBuff` 为空时，技能不附加 Buff
- `SkillList` 为空时，怪物不放技能但仍可移动和普攻
- `RedDot` 未配置时，相关入口不显示红点

## 4. 配置检查顺序

建议固定按以下顺序检查：

1. 表结构是否存在
2. 表头字段是否一致
3. 主键是否重复
4. 必填字段是否为空
5. 枚举字段是否合法
6. 外键是否能引用到目标表
7. 数值范围是否越界
8. 运行时是否还能正常被读取

## 5. 表级检查规则

### 5.1 MapData

必须检查：

- `Id` 唯一
- `Name` 与场景/资源命名一致
- `Birth` 不为空
- `PlatformDataId` 引用有效

容错建议：

- `Music` 为空时使用默认音乐或静音
- `PlatformDataId` 为空时视为无动态平台地图

### 5.2 MainChapter

必须检查：

- `Id` 唯一
- `MapId` 可引用到 `MapData`
- 章节入口可被 UI 找到

### 5.3 MonsterSpawner

必须检查：

- `MapId` 存在
- `MonsterId` 可引用到 `Monster`
- `TriggerX / SpawnX / SpawnY` 合法
- `MaxCount` 大于 0

### 5.4 Monster

必须检查：

- `AiGroupId` 可引用到 `AIGroupDef`
- `SkillList` 中每个技能存在
- 属性值不为非法负数

### 5.5 AIGroupDef / AIActionDef

必须检查：

- `AiActionList` 不引用不存在的动作
- `Conditions` 和 `Actions` 可被运行时解析
- 动作时长和条件值合法

### 5.6 SkillDef / TimeLineDef / BulletDef

必须检查：

- `SkillDef.TimelineId` 存在
- `TimeLineDef.TimeLineParams` 可解析
- `BulletDef` 引用可用

容错建议：

- 缺少表现资源时保留逻辑效果
- 时间线非法时应直接阻断技能释放

### 5.7 UI 表

必须检查：

- `GameModule` 模块名唯一
- 入口控制器存在
- 打开条件可解析

## 6. 日志规范

每个配置错误日志至少包含：

- 表名
- 记录 ID
- 字段名
- 错误原因
- 建议修复方式

建议格式：

```text
[ConfigError] Table=MonsterSpawner Id=3 Field=MonsterId Reason=Missing reference Monster(1001)
```

## 7. 自动开发规则

AI 在处理配置时必须遵守：

- 不要为了继续开发而假设缺失字段的值。
- 不要擅自改表结构来绕过报错。
- 遇到阻断级错误先返回问题清单，再继续下一项。
- 如果表之间存在冲突，以 `01_约束_AI开发交接说明.md` 为准。

## 8. 最小验收

导表完成后，至少满足：

- 无重复 ID
- 无缺失必填字段
- 无非法外键
- 无枚举解析错误
- 关键战斗链路可正常读取配置
