# 游戏叙事设计与对话插入指南

本文档说明如何将背景故事和世界观融入三个章节的游戏流程中。

---

## 1. 叙事框架

### 游戏的世界观要点

**时间**：公元 2088 年  
**背景**：赛博朋克城市"霓虹之城"，企业秩序崩坏中  
**主角**：Zero（半机械人执行者，记忆受损）  
**冲突**：企业秩序 vs 黑客自由 vs AI 自主

**四个主要角色**：
- **Zero**：玩家操控的主角
- **Vanta**：黑客联络员（友方，提供支持）
- **Archon**：失控的 AI（神秘反派）
- **Helix Director**：企业安全官（任务下达者）

---

## 2. 对话和故事的插入位置

### 2.1 游戏启动到登录界面

**位置**：启动画面 / 过场动画

**内容**：
```
[黑屏，霓虹文字闪烁]

字幕：2088 年，霓虹城陷入协议冲突

[画面逐渐亮起，展示城市全景]

Helix Director（企业安全官）的声音：
"Zero，你的权限已被降格。
 现在你唯一的价值是'清理'失控节点。
 完成任务，恢复权限。"

[音效：警报声、数据扫描声]

Zero 的意识碎片：
"我...是谁？为什么我...记得...和平的记忆..."

[进入登录界面]
```

**作用**：
- 交代背景
- 建立主角身份
- 设置主要冲突

---

### 2.2 进入游戏前（预加载流程）

**位置**：加载界面，或"开始游戏"点击后

**内容**：
```
[简短的数据加载动画，同时有 Vanta 的声音]

Vanta（黑客组织）：
"Zero，我是 Vanta。企业对你的记忆封存还没解开。
 我在频率里埋了一些数据碎片。
 等你清理完这层，也许能找到真相。
 
 首先，去下城贫民窟。
 那里有失控的感染体，企业想全部清除。
 但...你需要知道，他们不全是'病毒'。"

Zero：
"...我接受任务。"

```

**作用**：
- 介绍友方角色
- 交代主角目标（完成任务 → 恢复权限）
- 埋下悬念（失控体不是病毒？）

**章节主题**：

**DialogueData.xlsx**：

| Id | Chapter | NodeName | Speaker | Text | Type | NextNodeId | Condition |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 1 | Intro | StartGame | Helix | Zero，你的权限已被降格... | 自动 | 2 | 无 |
| 2 | Intro | CharacterIntro | Zero | 我...是谁？ | 自动 | - | 无 |

**字段含义**：
- `Speaker`：角色名（Helix, Vanta, Archon, Zero）
- `Type`：对话类型（自动自动播放，或需要用户选择）
- `NextNodeId`：下一个节点（自动顺序或分支）
- `Condition`：触发条件（如玩家位置、击杀数量、权限等级）

### 5.3 代码实现框架

**DialogueManager.cs**：

```csharp
public class DialogueManager : MonoBehaviour
{
    // 播放对话
    public void PlayDialogue(int dialogueId)
    {
        var dialogueData = GetDialogueData(dialogueId);
        
        // 显示对话 UI
        dialoguePanel.Show(dialogueData.Speaker, dialogueData.Text);
        
        // 如果是选择对话，显示选择按钮
        if (dialogueData.Type == DialogueType.Choice)
        {
            ShowChoices(dialogueData.ChoiceIds);
        }
        else
        {
            // 自动播放，延迟后进入下一个
            StartCoroutine(AutoPlayNext(dialogueData.NextNodeId));
        }
    }
    
    // 选择回调
    public void OnChoiceSelected(int choiceId)
    {
        // 根据选择，记录玩家的伦理倾向
        RecordPlayerChoice(choiceId);
        
        // 继续下一个对话
        PlayDialogue(GetNextNodeId(choiceId));
    }
}
```

**DialogueTrigger.cs**：

```csharp
public class DialogueTrigger : MonoBehaviour
{
    public int dialogueId;
    public DialogueCondition condition;
    
    private bool triggered = false;
    
    public void Update()
    {
        // 检查触发条件
        if (!triggered && condition.IsMet())
        {
            DialogueManager.Instance.PlayDialogue(dialogueId);
            triggered = true;
        }
    }
}
```

## 6. 对话与玩法的融合

### 6.1 对话不应该打断游戏体验

**错误做法**：
```
玩家正在紧张的战斗中
→ 突然冒出长段对话
→ 玩家被打败了，怪"为什么要插话"
```

**正确做法**：
```
玩家击杀一波怪物，短暂休息
→ Vanta 说一句话（2-3 秒）
→ 玩家思考，进入下一波

或者

战斗结束，玩家到达关卡节点
→ 自动触发过场动画 + 对话
→ 玩家做出选择或被告知故事发展
```

**对话 UI 设计**：

## 7. 对话的伦理选择系统

### 选择点设计

三个关键选择点决定最终结局：

| 选择点 | 问题 | A 选项 | B 选项 | C 选项 |
| :--- | :--- | :--- | :--- | :--- |
| **Ch1 中点** | 对失控体的态度 | 无情清理 | 怜悯保护 | - |
| **Ch2 结束** | 对 Archon 的态度 | 摧毁病毒 | 同情生命 | - |
| **Ch3 Boss** | 最终选择 | 维护秩序 | 支持 Archon | 寻找平衡 |

### 选择的影响

```
选择记录系统：

玩家选择A（清理、摧毁、秩序）×3 → 结局 A：企业秩序维持
玩家选择B（保护、同情、自由）×3 → 结局 B：黑客自由胜利
玩家混合选择或选C → 结局 C：新的平衡（隐藏结局）

中间阶段评价：
- 权限恢复 60-100%（基于任务完成度）
- 伦理倾向评分（基于选择）
- 可解锁的额外对话（不同伦理的暗线故事）
```

```
NodeId: string (唯一标识，如 "Ch1_MidPoint")

Text: string (对话内容，支持多行)
TextZh: string (中文翻译，如果需要多语言)
VoiceId: int (语音资源ID，0 表示无语音)
VoiceActor: string (语音演员名)
Type: enum (Auto 自动播放, Choice 选择, Cutscene 过场)
Duration: float (自动播放时停留时间，秒)
NextNodeId: int (下一个对话ID，Auto 类型用)
ChoiceIds: string (选择的分支IDs，逗号分隔，Choice 类型用)
ChoiceTexts: string (选择按钮文字，逗号分隔)
TriggerCondition: string (触发条件，如 "X>3&Wave1Complete")
CameraShake: bool (是否摄像机震动)
SoundEffect: string (声效资源名)
Subtitle: bool (是否显示字幕)
Timestamp: float (在关卡中触发的时间或位置)
```
