# UI 交互模块开发规范

本文档用于补足 `Work/12_设计_UI和UX交互规范.md` 的落地层，说明 UI 模块如何接入、如何组织代码，以及交互事件该怎么写。

## 1. 适用范围

适用于当前工程里基于 `ExPanel + UIModuleManager + UIContainer` 的 UI 模块。

当前已存在的典型模块：
- `Login`
- `FightMainChapter`
- `FightShare`

## 2. 当前架构约定

### 2.1 模块入口

`Assets/HotAssets/Scripts/UI/Core/UIModuleManager.cs`

职责：
- 读取 `TbGameModule`
- 注册模块
- 初始化 `Controller / Model / View`
- 打开和判断模块入口

### 2.2 面板基类

`Assets/HotAssets/Scripts/UI/Tool/Component/ExPanel.cs`

职责：
- 管理打开 / 关闭
- 处理层级、遮罩、动画、子 UI
- 提供 `RegisterUI()` 和 `RegisterEvent()` 扩展点

### 2.3 数据与表现分层

- `Controller`：处理事件、协议、模块跳转
- `Model`：组织和缓存数据
- `View`：只负责表现和界面绑定

禁止：
- 在 `View` 里写复杂业务逻辑
- UI 直接计算战斗规则
- 模块之间用硬编码对象直接互调

## 3. 新模块开发步骤

### 3.1 建目录

建议按现有结构创建：

- `Assets/HotAssets/Scripts/UI/Module/<ModuleName>/`
- `Assets/HotAssets/UI/Prefabs/<ModuleName>/`

### 3.2 建脚本

通常至少包含：

- `<ModuleName>Panel.cs`
- `<ModuleName>Controller.cs`
- `<ModuleName>Model.cs`
- `<ModuleName>View.cs`
- `<ModuleName>Const.cs`

### 3.3 接入管理器

新增模块后，需要同步：

- `ModuleType` 枚举
- `TbGameModule` 数据
- `UIModuleManager.RegisterAll()`
- 必要时补 `CheckModuleOpenCondition()`

### 3.4 绑定逻辑

面板里建议遵循：

- `OnInit()` 做节点缓存和初始监听
- `RegisterUI()` 做子节点绑定
- `RegisterEvent()` 做事件订阅
- `OnOpen()` 做刷新和数据拉取
- `OnClose()` 做反注册和清理

## 4. 交互规则

### 4.1 按钮

按钮至少要满足：

- 点击有反馈
- 重复点击有去重或禁用态
- 失败时给出明确提示

### 4.2 事件

建议通过事件驱动刷新 UI：

- 逻辑层发事件
- UI 层订阅事件
- UI 只做展示，不反向持有规则

### 4.3 异步资源

如果界面依赖异步加载：

- 先显示加载态
- 加载失败要有兜底提示
- 关闭界面时要清理已加载资源

## 5. 现有案例

### 5.1 登录页

`Assets/HotAssets/Scripts/UI/Module/Login/LoginPanel.cs`

特点：
- `ExButton` 通过 `PanelUIContainer.Get<ExButton>("m_StartGame")` 获取
- 点击后直接进入战斗入口

### 5.2 战斗通用 UI

`Assets/HotAssets/Scripts/UI/Module/FightShare/OverlayFightHudUI.cs`

特点：
- 订阅战斗事件
- 运行时实例化共享 HUD prefab
- 通过 `UIContainer` 读取节点

## 6. 验收清单

- 面板能正常打开和关闭
- 重复打开不报错、不残留旧状态
- 按钮点击后有明确反馈
- 事件刷新逻辑正确
- 异步加载失败有提示
- 新模块已完成注册和入口配置
