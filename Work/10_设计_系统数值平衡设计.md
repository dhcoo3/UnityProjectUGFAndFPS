# 1. 核心数值与系统平衡设计文档 (System & Balance Design)

## 1.1 文档目标

本文档用于定义项目的数值基线、战斗节奏和成长节奏，供以下工作统一使用：

- 战斗逻辑开发
- 配置表字段设计
- 关卡难度编排
- 掉落与经济投放
- QA 平衡回归

本文档中的数值均以“首个可玩版本”为目标，不追求一次定终版，优先满足以下原则：

- 同类武器之间有明确定位差异
- 玩家操作收益高于纯属性碾压
- 怪物威胁来自行为和组合，不只来自血量膨胀
- 局内战斗资源和局外养成资源彼此独立，避免系统缠绕

## 1.2 核心战斗体验目标

首个可玩版本的战斗体验目标如下：

- 平均战斗节奏为“高机动 + 高频普通攻击 + 中频技能爆发”
- 玩家能通过走位、跳跃、闪避和打断来降低受伤，而不是单纯堆数值
- 普通敌人击杀时间控制在 `1.0s - 3.5s`
- 精英敌人击杀时间控制在 `8s - 20s`
- Boss 战单场时长控制在 `2min - 4min`

体验判定标准：

- 玩家普通战斗中每 `8s - 15s` 至少发生一次明确决策
- 单次受击应让玩家感知风险，但不应频繁一击致死
- 武器更换后应在 `30s` 内感知到显著手感差异

## 1.3 战斗数值模型

项目战斗逻辑基于定点数 `fix / fix2 / fix3` 实现，表现层不得使用 `float` 结果反推逻辑。

### 1.3.1 伤害结算

建议统一采用如下结构：

- `RawDamage = BaseDamage + AttackScale * AttackerAttack`
- `DefenseRate = TargetDefense / (TargetDefense + K)`
- `CritMultiplier = 1` 或 `CritDamage`
- `FinalDamage = RawDamage * (1 - DefenseRate) * CritMultiplier * ElementModifier * SkillModifier`

建议首版常量：

- 防御系数 `K = 800`
- 默认暴击伤害倍率 `CritDamage = 1.5`
- 默认暴击率上限不超过 `60%`

设计说明：

- 防御采用收益递减结构，避免“高防完全无效”或“高防完全无敌”
- 暴击作为爆发放大器，不作为常驻主收益来源
- 元素修正与技能修正应后乘，便于按系统分开调试

### 1.3.2 命中与碰撞

- 本项目采用 HitBox/碰撞体判定，不使用命中率与闪避率随机判定
- 命中成立条件由逻辑层碰撞结果决定
- 表现层只能消费 `命中成功`、`命中失败`、`穿透`、`格挡` 等事件

### 1.3.3 击退与硬直

建议使用三个参数描述控制手感：

- `HitStunFrame`：受击硬直帧数
- `KnockBackForce`：击退力度
- `Weight`：目标重量

建议计算：

- `KnockBackDistance = KnockBackForce / max(Weight, 1) * Attenuation`

首版经验值：

- 普通小怪：`Weight = 80 - 120`
- 近战精英：`Weight = 150 - 220`
- Boss：`Weight = 300+`

## 1.4 玩家属性体系

### 1.4.1 核心属性

建议首版统一采用以下核心属性：

- `MaxHP`：生命上限
- `Attack`：攻击力
- `Defense`：防御力
- `CritRate`：暴击率
- `CritDamage`：暴击伤害倍率
- `MoveSpeed`：移动速度
- `JumpPower`：跳跃力度
- `EnergyMax`：技能资源上限
- `EnergyRecover`：技能资源回复速度

其中：

- `MoveSpeed`、`JumpPower` 影响操作体验，策划调整时必须经过关卡复测
- `CritRate`、`CritDamage` 属于进阶收益，不建议在前两章过早堆高
- `Energy` 系统用于限制技能刷屏，保证普攻与技能节奏均有存在感

### 1.4.2 等级成长

- 等级上限：`60`
- 建议成长曲线：`Value = Base * (1 + GrowthRate)^(Level - 1)`
- 首版成长区间建议控制在 `每 10 级总战力增长 18% - 28%`

不建议出现的情况：

- 低级怪因等级压制完全打不动玩家
- 升级前后差距不明显，玩家感知不到成长
- 所有成长都集中在攻击力，导致玩法单一

### 1.4.3 首版玩家基准值

以下为首版测试基准，不是最终发布值：

| 项目 | 建议值 | 说明 |
| :--- | :--- | :--- |
| 初始生命 | 1000 | 保证前两章容错 |
| 初始攻击 | 100 | 便于计算武器倍率 |
| 初始防御 | 80 | 避免小怪瞬秒 |
| 暴击率 | 5% | 初版低暴击基线 |
| 暴伤倍率 | 150% | 与大多数技能爆发兼容 |
| 移动速度 | 1.0 基准 | 由角色动作系统映射 |
| 能量上限 | 100 | 便于技能资源显示 |

## 1.5 武器定位与差异化

武器不应只靠面板高低区分，必须同时在射击节奏、有效距离、位移风险、后摇管理上体现差异。

### 1.5.1 武器定位表

| 武器类型 | 射速 | 单发伤害 | 射程 | 弹匣 | 换弹 | 核心定位 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 电磁手枪 | 中 | 中 | 中 | 12 | 快 | 通用、容错高 |
| 脉冲步枪 | 高 | 中低 | 长 | 30 | 中 | 持续压制 |
| 重型霰弹 | 低 | 极高 | 短 | 5 | 慢 | 爆发、击退 |
| 轨道狙击 | 极低 | 极高 | 极长 | 3 | 慢 | 远点杀伤、惩罚失误 |

### 1.5.2 设计规则

- 手枪是默认基准，不允许任何场景完全淘汰手枪
- 步枪负责稳定输出，适合作为主线默认毕业武器
- 霰弹通过近距离高收益换取高风险，不能无代价清场
- 狙击必须依赖拉距和预判，近身场景应显著吃亏

### 1.5.3 手感指标

每把武器至少定义以下参数：

- `FireInterval`
- `ReloadTime`
- `BulletSpeed`
- `Spread`
- `HitStopFrame`
- `KnockBackForce`
- `AmmoMax`
- `PenetrationCount`

策划验收时必须同时看：

- 面板 DPS
- 实战 DPS
- 空枪惩罚
- 换弹中断成本
- 小怪清理效率
- 精英压制能力

## 1.6 技能与资源节奏

### 1.6.1 技能类型划分

首版建议至少有以下技能类型：

- 位移技：用于拉扯、躲招、切入
- 爆发技：短时间高伤害
- 控制技：击退、减速、短硬直
- 生存技：护盾、减伤、回复

### 1.6.2 冷却与资源建议

| 类型 | 冷却时间 | 资源消耗 | 设计目的 |
| :--- | :--- | :--- | :--- |
| 位移技 | 4s - 8s | 低 | 维持高机动 |
| 爆发技 | 10s - 18s | 中 | 提供战斗节奏峰值 |
| 控制技 | 8s - 15s | 中 | 用于拆解敌群 |
| 生存技 | 15s - 30s | 中高 | 作为容错工具 |

规则：

- 所有技能必须存在明确窗口期和代价
- 不能出现无脑循环释放即可覆盖全部战斗的技能
- 技能数据应尽量落到 `TbSkillDef`、`TbTimeLineDef`、`TbBuffDef`、`TbAoeDef`

## 1.7 敌人与难度曲线

### 1.7.1 敌人分类

首版怪物建议按功能划分，而不是只按血量强弱划分：

- `Pressure`：高速近战，逼迫玩家移动
- `Shooter`：远程火力，迫使玩家找掩体或跳台
- `Tank`：高血高硬直抗性，拖长战线
- `Support`：护盾、召唤、增益
- `Boss`：多阶段机制压制

### 1.7.2 基础数值区间

| 敌人类型 | 生命系数 | 攻击系数 | 防御系数 | 重量系数 |
| :--- | :--- | :--- | :--- | :--- |
| 普通近战 | 1.0 | 1.0 | 1.0 | 1.0 |
| 普通远程 | 0.8 | 1.1 | 0.8 | 0.8 |
| 精英坦克 | 2.5 | 1.2 | 1.6 | 1.8 |
| 精英输出 | 1.5 | 1.8 | 1.0 | 1.1 |
| Boss | 8.0+ | 2.0+ | 2.0+ | 3.0+ |

说明：

- 普通远程威胁主要来自站位和火力线，不来自坦度
- 精英坦克要通过压迫感制造决策成本，而不是单纯耗时
- Boss 必须依赖机制分段，不能只是“超大血条小怪”

## 1.8 章节难度目标

建议主线首版按 3 个章节构建：

| 章节 | 目标时长 | 目标主题 | 核心难点 |
| :--- | :--- | :--- | :--- |
| 第一章 | 8 - 10 分钟 | 教学与基础移动 | 熟悉移动、跳跃、普通攻击 |
| 第二章 | 10 - 12 分钟 | 多波次与高压远程 | 学会找站位与技能拆场 |
| 第三章 | 12 - 15 分钟 | 机制 Boss 与资源管理 | 学会保技能、处理多阶段压力 |

章节设计规则：

- 每章至少引入一个新敌人行为
- 每章至少验证一个新系统是否成立
- 难度上升优先来自“组合复杂度”，其次才是数值提升

## 1.9 经济与掉落设计

### 1.9.1 核心货币

- 主货币：`Cyber Credits`
- 技能强化材料：`Bit-Core`
- 稀有升级材料：`Protocol Shard`

### 1.9.2 投放占比建议

- 关卡直接产出：`60% - 70%`
- 主线任务奖励：`15% - 20%`
- 周期目标或成就奖励：`10% - 15%`
- 分解与回收：`5% - 10%`

### 1.9.3 消耗占比建议

- 武器强化：`40% - 50%`
- 技能升级：`25% - 35%`
- 消耗品与商店：`10% - 20%`
- 角色属性解锁：`10% - 15%`

规则：

- 任何单一系统都不应吞掉全部资源
- 前期玩家不应因资源过卡导致无法推进主线
- 稀有材料应主要作为“阶段门槛”，不作为日常高频消耗

## 1.10 平衡调优流程

数值调优必须遵循以下顺序：

1. 先定武器与敌人的功能定位
2. 再定基础数值区间
3. 再定章节投放与掉落
4. 最后做高阶成长和稀有收益

调优时禁止：

- 直接通过增加血量掩盖敌人机制缺失
- 在没有战斗日志和实战回放的情况下频繁改公式
- 玩家太强时只砍伤害，不看资源、站位和技能循环

## 1.11 平衡验收指标

每次平衡版本至少记录以下指标：

- 普通怪平均击杀时间
- 精英怪平均击杀时间
- Boss 战平均通关时长
- 玩家平均受击次数
- 玩家技能使用频率
- 主武器命中率
- 平均关卡死亡次数
- 资源净增与净耗

验收结论分为：

- 可用：体验稳定，允许继续做内容
- 待调：能玩，但存在明显单解或断层
- 不通过：存在压倒性最优、资源崩坏或难度断崖

## 1.12 当前实体配置表字段映射

以下内容基于当前工程中生成出的 `cfg` 字段，不是抽象建议字段。

### 1.12.1 `Hero`

当前 `Hero` 实际字段包括：

| 字段 | 含义 | 用途 | 风险点 |
| :--- | :--- | :--- | :--- |
| `Id` | 角色编号 | 角色唯一标识 | 不可重复 |
| `Name` | 名称 | UI 和调试显示 | 不能只写测试名 |
| `Asset` | 资源路径 | 角色资源加载 | 资源丢失会导致创建失败 |
| `AnimGroup` | 动画组 | 动画映射 | 与美术资源强绑定 |
| `MoveSpeed` | 初始移动速度 | 角色移动手感 | 调太高会破坏关卡跳跃 |
| `Hp` | 初始血量 | 生存能力 | 与 QA 容错测试强相关 |
| `Ammo` | 弹仓/技能资源 | 技能与射击消耗 | 为 0 时需验证是否还能开局战斗 |
| `Attack` | 攻击力 | 伤害计算 | 会影响全部武器收益 |
| `ActionSpeed` | 行动速度 | 动画和时间线整体速度 | 高值易导致表现错位 |
| `BodyRadius` | 体型碰撞半径 | 地图碰撞 | 过大易卡地形 |
| `HitRadius` | 受击半径 | 子弹命中判定 | 过大/过小都会破坏公平性 |
| `Stamina` | 耐力 | 位移和技能资源 | 需和冲刺技能联调 |
| `WallSlideSpeed` | 滑墙速度 | 墙体移动手感 | 需和关卡墙面设计一起验证 |
| `WallJumpForceY` | 蹬墙跳竖向力 | 墙跳高度 | 影响平台可达性 |
| `WallJumpForceX` | 蹬墙跳横向力 | 墙跳距离 | 影响越障能力 |
| `WallJumpDuration` | 墙跳水平持续时长 | 墙跳手感 | 过长会变相冲刺 |

建议：

- `MoveSpeed`、`WallJumpForceY`、`WallJumpForceX` 调整时必须回归第一章和第二章关卡
- `ActionSpeed` 不要作为单纯强度加成随意提高，它会放大动画和时间线节奏问题

### 1.12.2 `Monster`

当前 `Monster` 实际字段包括：

| 字段 | 含义 | 用途 | 风险点 |
| :--- | :--- | :--- | :--- |
| `Id` | 怪物编号 | 唯一标识 | 不可重复 |
| `Name` | 名称 | 调试与文案显示 | 建议统一命名风格 |
| `Desc` | 描述 | 策划和调试备注 | 不要留空关键怪 |
| `Asset` | 资源 | 角色实例化 | 丢资源会直接报错 |
| `AnimGroup` | 动画组 | 动作播放 | 与受击、死亡表现相关 |
| `AI` | AI 组编号 | 行为逻辑入口 | 配错会站桩或异常 |
| `SearchRange` | 索敌范围 | 追击和发现玩家 | 需和关卡视野统一 |
| `SkillList` | 技能列表 | 怪物技能加载 | 空列表会导致只会移动 |
| `MoveSpeed` | 移动速度 | 追击与巡逻 | 影响压力感 |
| `Hp` | 初始血量 | 生存能力 | 不能代替机制设计 |
| `Ammo` | 资源 | 技能释放 | 怪物可做技能资源限制 |
| `Attack` | 攻击力 | 伤害强度 | 与章节难度挂钩 |
| `ActionSpeed` | 行动速度 | 动作与时间线节奏 | 过高会引发表现跟不上 |
| `BodyRadius` | 体型半径 | 地形碰撞 | 大怪易卡边角 |
| `HitRadius` | 受击半径 | 命中判定 | 需与体型观感一致 |
| `Stamina` | 耐力 | 技能和位移资源 | 目前需按实际行为测试 |

怪物配置最低闭环：

1. `Monster.Id`
2. `Monster.Asset`
3. `Monster.AnimGroup`
4. `Monster.AI`
5. `Monster.SkillList`
6. `MonsterSpawner.MonsterId`

## 1.13 当前技能配置链路字段映射

### 1.13.1 技能总链路

当前技能链路的真实关系是：

`SkillDef -> TimeLineDef -> 时间线节点`

其中时间线节点可进一步引用：

- `SkillBulletDef -> BulletDef`
- `SkillAoeDef -> AoeDef`
- `BuffDef`

这意味着一个技能通常不是单表完成，而是由 3 到 6 张表共同定义。

### 1.13.2 `SkillDef`

| 字段 | 含义 | 说明 |
| :--- | :--- | :--- |
| `Id` | 技能编号 | 技能入口主键 |
| `Cost` | 资源消耗 | `Hp / Ammo / Stamina` |
| `Condition` | 使用条件 | 低于条件不可释放 |
| `TimeLine` | 时间线编号 | 技能行为主驱动 |
| `AddBuff` | 附加 Buff 列表 | 释放技能时附加 |

适用场景：

- 普攻
- 技能
- 冲刺
- 二段跳
- 怪物技能

### 1.13.3 `Resource`

当前技能资源结构只有三项：

| 字段 | 含义 |
| :--- | :--- |
| `Hp` | 血量消耗或门槛 |
| `Ammo` | 弹药/能量消耗或门槛 |
| `Stamina` | 耐力消耗或门槛 |

配置建议：

- 普攻一般走低 `Ammo` 消耗或不消耗
- 位移技优先消耗 `Stamina`
- 高风险技能可同时设置 `Condition` 和 `Cost`

### 1.13.4 `TimeLineDef`

| 字段 | 含义 | 说明 |
| :--- | :--- | :--- |
| `Id` | 时间线编号 | 与技能绑定 |
| `TimeLineParams` | 节点列表 | 事件按时间顺序执行 |
| `Duration` | 持续时间 | 决定技能完整时长 |

当前时间线节点类型包括：

- `SetCasterControlState`
- `CasterPlayAnim`
- `PlaySightEffectOnCaster`
- `FireBullet`
- `UseAoeToPoint`
- `PlaySound`
- `ApplyForceToCaster`

### 1.13.5 关键时间线节点

| 节点类型 | 当前字段 | 作用 |
| :--- | :--- | :--- |
| `SetCasterControlState` | `CanMove / CanRotate / CanUseSkill` | 控制施法期间输入锁定 |
| `CasterPlayAnim` | `AnimType / UseCurrentDeg` | 播放动作 |
| `PlaySightEffectOnCaster` | `BindPointKey / EffectName / EffectKey / Loop` | 播放角色身上特效 |
| `FireBullet` | `BulletDataId` | 发射子弹 |
| `UseAoeToPoint` | `SkillAoeId` | 生成范围效果 |
| `PlaySound` | `SoundName / Only / Interval` | 播放音效 |
| `ApplyForceToCaster` | `Speed / Duration / CanMoveInput / VerticalSpeed` | 冲刺、突进、跳跃推力 |

配置规则：

- 动作开始时优先配 `SetCasterControlState + CasterPlayAnim`
- 发射武器必须通过 `FireBullet`
- 范围技能必须通过 `UseAoeToPoint`
- 位移类技能优先通过 `ApplyForceToCaster`

### 1.13.6 `SkillBulletDef` 与 `BulletDef`

`SkillBulletDef` 负责“这次技能发出的子弹长什么样”，`BulletDef` 负责“这个子弹自身怎么运行”。

`SkillBulletDef` 当前字段：

| 字段 | 含义 |
| :--- | :--- |
| `Id` | 编号 |
| `BulletData` | 关联 `BulletDef` |
| `FirePos` | 发射偏移 |
| `Degree` | 发射角 |
| `Speed` | 初速度 |

### 1.13.8 `BuffDef`

`BuffDef` 当前结构已经比较完整，既支持状态控制也支持属性修改。

关键字段：

| 字段 | 含义 |
| :--- | :--- |
| `Tag` | Buff 标签 |
| `Priority` | 优先级 |
| `Permanent` | 是否永久 |
| `Duration` | 持续时间 |
| `MaxStack` | 最大层数 |
| `TickTime` | 周期触发 |
| `OnOccur / OnRemoved / OnTick` | 生命周期事件 |
| `OnCast / OnHit / BeHurt / OnKill / BeKilled` | 战斗事件挂钩 |
| `BuffAddState` | 移动/转向/施法限制 |
| `BuffPropMod` | 属性修改 |

常见用途：

- 冲刺期间禁转向或禁施法
- 中毒、燃烧等周期伤害
- 攻击加成、移速加成
- 受击硬直免疫或霸体

## 1.14 当前表驱动设计的最小落地模板

### 1.14.1 新增一个普通射击技能

最少需要配置：

1. `SkillDef`
2. `TimeLineDef`
3. `SkillBulletDef`
4. `BulletDef`

推荐顺序：

1. 先建 `BulletDef`
2. 再建 `SkillBulletDef`
3. 再建 `TimeLineDef`，用 `FireBullet`
4. 最后建 `SkillDef`

### 1.14.2 新增一个范围技能

最少需要配置：

1. `SkillDef`
2. `TimeLineDef`
3. `SkillAoeDef`
4. `AoeDef`

如需附加状态，再补：

5. `BuffDef`

### 1.14.3 新增一个怪物技能

最少需要配置：

1. `SkillDef`
2. `TimeLineDef`
3. 视技能形式接 `SkillBulletDef` 或 `SkillAoeDef`
4. 把技能 ID 放进 `Monster.SkillList`
5. 保证 `Monster.AI` 对应行为会调用该技能
