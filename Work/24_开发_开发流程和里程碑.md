## 3. 配置表依赖关系图

```
配置表之间的引用关系：

MainChapter (主入口)
    │
    └─ MapId ──→ MapData (地图定义)
                  │
                  ├─ PlatformDataId ──→ PlatformDef[] (移动平台)
                  │
                  └─ 场景加载 ──→ 碰撞文件 (ChapterXX_MapXX.bytes)


MonsterSpawner (刷怪配置)
    │
    ├─ MonsterId ──→ Monster (怪物属性)
    │                   │
    │                   ├─ AiGroupId ──→ AIGroupDef (AI组)
    │                   │                   │
    │                   │                   └─ ActionIdList ──→ AIActionDef[] (AI行为)
    │                   │                                           │
    │                   │                                           └─ SkillId ──→ SkillDef (技能)
    │                   │
    │                   └─ SkillList ──→ SkillDef[] (技能)
    │
    └─ SpawnX/Y (刷新位置参考)


SkillDef (技能)
    │
    └─ TimelineId ──→ TimeLineDef (时间线)
                       │
                       └─ TimeLineParams (事件数组)
                           │
                           ├─ FireBullet ──→ SkillBulletDef
                           │                  │
                           │                  └─ BulletId ──→ BulletDef
                           │
                           ├─ UseAoeToPoint ──→ SkillAoeDef
                           │                     │
                           │                     └─ AoeDataId ──→ AoeDef
                           │
                           └─ ApplyForceToCaster (位移)


BuffDef (可选)
    │
    └─ 在 SkillDef 的 AddBuff 中引用
```

## 5. 关键控制点 (Milestone)

```
M1: 第一章进图闭环 ✅
    ├─ 能从登录进入第一章
    ├─ 玩家可操作（移动、跳跃、射击）
    ├─ 怪物按时刷新并可被击杀
    └─ 完整结算流程

M2: 第二章高低差验证 ✅
    ├─ 移动平台正常工作
    ├─ 远程怪物有压制能力
    ├─ 玩家技能能有效拆场
    └─ 无地形卡死

M3: 第三章 Boss 战完成 ✅
    ├─ Boss 多技能正常释放
    ├─ 胜负判断和结算正确
    ├─ 能返回主菜单或重新挑战
    └─ 三章主线可玩版本成立

M4: 优化与发布准备 🔜
    ├─ 性能优化
    ├─ 热更新流程验证
```
