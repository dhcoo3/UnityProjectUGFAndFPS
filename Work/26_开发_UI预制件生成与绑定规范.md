# UI 预制件生成与绑定规范

本文档用于说明 UI Prefab 的创建、命名、绑定和复用方式，补足当前工程里“脚本有了，但 prefab 该怎么落地”的部分。

## 1. 适用范围

适用于 `Assets/HotAssets/UI/Prefabs/` 下所有 UI 预制件。

当前可见结构示例：

- `Assets/HotAssets/UI/Prefabs/Login/LoginPanel.prefab`
- `Assets/HotAssets/UI/Prefabs/FightCommon/OverlayFightHudUI.prefab`
- `Assets/HotAssets/UI/Prefabs/FightCommon/OverlayFightDamageUI.prefab`
- `Assets/HotAssets/UI/Prefabs/FightCommon/RoleHud.prefab`

## 2. 目录约定

建议按模块分目录：

- `Login`
- `FightMainChapter`
- `FightCommon`

其中：
- 入口面板放模块目录
- 通用战斗 UI 放 `FightCommon`
- 可复用子面板单独放一份 prefab

## 3. Prefab 创建标准

### 3.1 根节点

每个 prefab 根节点建议与面板脚本同名，例如：

- `LoginPanel`
- `FightMainChapterPanel`
- `OverlayFightHudUI`

### 3.2 必要组件

面板 prefab 根节点通常应具备：

- `UIContainer`
- `ExPanel` 对应脚本
- `Canvas` / `CanvasGroup` / `GraphicRaycaster`

这些组件多数由 `ExPanel` 运行时补齐，但 prefab 结构仍要保持干净、可读。

### 3.3 节点命名

绑定节点统一使用稳定命名，例如：

- `m_StartGame`
- `m_HpBarBg`
- `m_HpBar`
- `m_Name`

原则：
- 名称必须和脚本里获取的 key 完全一致
- 关键节点不要随意改名
- 复用节点优先保持同一套命名

## 4. 绑定流程

### 4.1 先搭 prefab 层级

先在 prefab 里把按钮、文本、图片、容器结构搭好。

### 4.2 再写 UIContainer

把需要脚本读取的节点挂进 `UIContainer`。

### 4.3 最后接脚本

脚本里通过以下方式绑定：

- `PanelUIContainer.Get<T>("节点名")`
- `GetComponent<UIContainer>()`
- `RegisterUI()` 里做二次绑定

### 4.4 典型写法

`LoginPanel`：
- `m_StartGame = PanelUIContainer.Get<ExButton>("m_StartGame")`

`FightUIRoleHud`：
- `m_HpBarBg = m_uiNode.Get<ExImage>("m_HpBarBg")`
- `m_HpBar = m_uiNode.Get<ExImage>("m_HpBar")`
- `m_Name = m_uiNode.Get<ExText>("m_Name")`

## 5. 预制件生成规则

### 5.1 新建面板 prefab

当新增 UI 模块时，先生成对应 prefab，再补脚本绑定。

### 5.2 新建子 UI prefab

如果界面中有可重复实例化的条目，建议单独做成子 prefab。

当前项目里类似：

- `RoleHud.prefab`
- `DamageTextUI.prefab`

### 5.3 运行时加载资源

如果 prefab 不是主界面常驻资源，而是运行时加载：

- 用 `AssetPathUtil.GetUIFormPath()` 统一路径
- 打开时加载
- 关闭时卸载

## 6. 资源与路径约定

UI 资源相关目录建议统一管理：

- `Assets/HotAssets/UI/Prefabs/`
- `Assets/HotAssets/UI/Sprites/`
- `Assets/HotAssets/UI/Atlas/`
- `Assets/HotAssets/UI/AtlasSprite/`

原则：
- prefab 路径要和模块名对得上
- 共享资源优先放公共目录
- 不要把同一套 UI 资源拆散到多个模块目录

## 7. 绑定校验

- prefab 根节点和脚本名一致
- `UIContainer` 里有完整节点
- 节点 key 和脚本 key 一致
- 按钮可点击
- 文本和图片能正确显示
- 运行时实例化的子 prefab 可正常释放

## 8. 常见问题

- 节点改名后脚本取不到值
- prefab 路径和资源表不一致
- 运行时实例化后没做卸载
- 共享 UI 误放到模块目录里，导致复用差
