# 配置表字段说明与导表指南

本文档合并配置表快速参考和配置说明，为策划和程序提供配置表字段的详细说明和导表工作流程。

本文档是当前三章闭环版本的配置基线。若与 `21_开发_12章配置表完整数据.md` 等扩展文档冲突，以本文档和 `01_约束_AI开发交接说明.md` 为准。

---

## 1. Luban 导表流程

### 1.1 导表基本步骤

1. 在 `Config/DataTables/Datas` 目录下维护所有 Excel 配置文件
2. 执行 `Config/gen.bat` 脚本
3. Luban 根据 `Config/DataTables/luban.conf` 配置生成：
   - C# 代码输出到 `Assets/HotAssets/Scripts/DataTable`
   - 二进制数据输出到 `Assets/HotAssets/DataTable`
4. 运行时通过 `AppEntry.DataTable.GetDataTableLuBan<T>()` 加载配置

### 1.2 导表检查清单

导表后务必检查以下项：

- [ ] gen.bat 执行成功，无 ERROR
- [ ] `Assets/HotAssets/Scripts/DataTable/` 中生成了新的 C# 代码
- [ ] `Assets/HotAssets/DataTable/` 中生成了二进制数据
- [ ] 所有外键都有效（如 MonsterSpawner.MonsterId 能引用 Monster.Id）
- [ ] 没有缺失必要字段的数据行

## 2. 配置表字段详解

### 2.1 MapData（地图基础数据）

| 字段名 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| Id | int | 地图唯一 ID，用于被 MainChapter 引用 | 101 |
| Name | string | 地图名称，用于美术资源查找 | ChapterO1_MapO1 |
| Random | bool | 是否随机生成（暂未实现） | false |
| Birth | Fix2 | 玩家出生点坐标 | (0, 1) |
| Music | string | 背景音乐资源路径 | "" |
| PlatformDataId | List<int> | 这个地图中使用的移动平台 ID 列表 | [] 或 [201, 202] |

### 2.2 MainChapter（主线章节）

| 字段名 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| Id | int | 章节唯一 ID，从 UI 到关卡的入口 | 1 |
| Name | string | 章节名称，显示在 UI 中 | 第一章：初入战场 |
| MapId | int | 这个章节使用的地图 ID | 101 |
| DungeonType | DungeonType | 副本类型，当前只有 Main | Main |

### 2.3 PlatformDef（移动平台）

| 字段名 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| Id | int | 平台唯一 ID，写入 MapData.PlatformDataId | 201 |
| Name | string | 平台名称 | 上升平台1 |
| StartX | Fix | 起点 X 坐标 | 3 |
| StartY | Fix | 起点 Y 坐标 | 8 |
| EndX | Fix | 终点 X 坐标 | 3 |
| EndY | Fix | 终点 Y 坐标 | 12 |
| Speed | Fix | 移动速度（单位/ms） | 40 |
| HalfWidth | Fix | 平台宽度的一半 | 1.5 |
| HalfHeight | Fix | 平台高度的一半 | 0.3 |
| AssetPath | string | 平台预制体路径 | Prefabs/Platform/MovingPlatform |

### 2.4 MonsterSpawner（怪物刷新点）

| 字段名 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| Id | int | 刷怪点唯一 ID | 1 |
| MapId | int | 这个刷怪点所在的地图 | 101 |
| MonsterId | int | 要刷什么怪，引用 Monster.Id | 1001 |
| TriggerX | Fix | 玩家到达这个 X 坐标时触发刷怪 | 3 |
| SpawnX | Fix | 怪物出生点 X 坐标 | 3 |
| SpawnY | Fix | 怪物出生点 Y 坐标 | 1 |
| PatrolMode | int | 巡逻方式（0=原地，1=来回） | 0 |
| PatrolCenterX | Fix | 巡逻中心 X 坐标 | 3 |
| PatrolHalfRange | Fix | 巡逻范围半径 | 2 |
| MaxCount | int | 同时存活的最多怪物数 | 2 |

### 2.5 Monster（怪物定义）

| 字段名 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| Id | int | 怪物唯一 ID | 1001 |
| Name | string | 怪物名称 | 小兵A |

### 2.6 AIGroupDef（AI 行为组）

| 字段名 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| Id | int | AI 行为组唯一 ID | 10 |
| Name | string | AI 行为组名称 | 小兵A基础AI |
| AiActionList | List<int> | 这个 AI 组包含的所有动作 ID | [100, 101, 102] |

### 2.7 AIActionDef（单个 AI 动作）

| 字段名 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| Id | int | 动作唯一 ID | 100 |
| Name | string | 动作名称 | 巡逻 |
| Duration | float | 这个动作最长执行时间（ms） | 5000 |
| Conditions | List<AICondition> | 执行这个动作的条件，如"距离玩家>100" | - |
| Actions | List<AIAction> | 执行的具体动作列表，如"SetMoveDir(1)" | - |

### 2.8 SkillDef（技能定义）

| 字段名 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| Id | int | 技能唯一 ID | 10001 |
| Name | string | 技能名称 | 普通射击 |
| Cost | Resource | 技能消耗资源，格式如 (ammo:1, stamina:0, hp:0) | (ammo:1) |
| CoolDown | Fix | 技能冷却时间（ms） | 300 |
| TimelineId | int | 这个技能的时间线 ID | 20001 |
| AddBuff | List<int> | 技能施加的 Buff 列表（可选） | [] |

### 2.9 TimeLineDef（技能时间线）

| 字段名 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| Id | int | 时间线唯一 ID | 20001 |
| Duration | Fix | 时间线总长度（ms） | 500 |
| TimeLineParams | List<TimeLineDefParam> | 时间线上的事件列表 | JSON 数组 |

TimeLineDefParam 的事件类型：
- `SetCasterControlState`：设置施法者控制状态
- `CasterPlayAnim`：播放施法者动画
- `PlaySightEffectOnCaster`：在施法者身上播放特效
- `FireBullet`：发射子弹
- `UseAoeToPoint`：在指定位置创建 AOE
- `PlaySound`：播放音效
- `ApplyForceToCaster`：给施法者施加力（如击退）

## 4. 常见配置错误与解决

### 4.1 外键错误

**问题**：导表时出现 "xxx field reference error"

**原因**：MonsterSpawner.MonsterId 引用了不存在的 Monster.Id

**解决**：
1. 检查 MonsterSpawner 中的 MonsterId 是否在 Monster 表中存在
2. 检查引用 ID 是否错误

### 4.2 缺失字段

**问题**：某些行导表失败

**原因**：必要字段为空或格式不正确

**解决**：
1. 检查所有必要字段是否填写
2. 检查数据格式是否匹配（如 Fix、List、Enum）

### 4.3 坐标越界

**问题**：怪物或平台出生点看起来不在地图内

**原因**：坐标超出地图边界

**解决**：
1. 核实 MapData 中的地图尺寸
2. 确保 MonsterSpawner 和 PlatformDef 的坐标在地图范围内

## 5. 配置填写工作流程

### 第一步：需求确认

定义以下内容：
- 新增关卡名称和内容
- 关卡内的怪物类型与数量
- 关卡的平台数量和配置
- 技能与 AOE 的表现

### 第二步：配置设计

1. 设计 MainChapter 记录
2. 设计 MapData 和 PlatformDef
3. 设计 MonsterSpawner 布置
4. 设计 Monster、AIGroupDef、AIActionDef
5. 设计 SkillDef、TimeLineDef、BulletDef、AoeDef

### 第三步：Excel 填写

1. 打开相应的 Excel 文件
2. 按照表设计逐一填写数据
3. 务必遵守数据格式（ID 唯一、外键引用正确）

### 第四步：导表

1. 执行 `Config/gen.bat`
2. 检查是否成功
3. 验证生成的 C# 代码

### 第五步：测试

1. 进入游戏加载配置
2. 进入关卡验证是否正确
3. 验证怪物刷新、AI 行为、技能释放等
