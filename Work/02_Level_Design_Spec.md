# 2. 玩法与关卡设计文档 (Level Design Spec)

## 2.1 关卡结构与元素
### 地图层级 (基于 Tilemap)
- **阻挡层 (Collision)**：物理碰撞，使用 `fix` 逻辑检测。
- **视觉层 (Visual)**：纯表现，不参与逻辑。
- **背景层 (Background)**：多层视差滚动 (Parallax Scrolling)。

### 交互机关
- **单向门**：只能从一侧打开。
- **激光陷阱**：周期性开启，定点数判定伤害。
- **移动平台**：循环移动，玩家站在上面会随之位移。
- **传送点**：副本波次切换的触发点。

## 2.2 刷怪逻辑与波次 (Wave Design)
- **触发方式**：
    - 进入区域触发 (Trigger Volume)。
    - 消灭上一波怪物后自动触发。
- **怪物组合 (Composition)**：
    - `坦克 + 远程`：常见组合，强制位移。
    - `高速近战 + 飞行`：干扰玩家走位。

## 2.3 关卡节奏规划 (Pacing)
1. **教学期**：简单的跳跃与基础射击。
2. **平稳期**：低密度的巡逻怪。
3. **小高潮**：封闭房间的大规模刷怪。
4. **Boss 战**：独特的机制对抗，具有明显的攻击前奏。

## 2.4 胜利与失败条件
- **胜利**：击败指定 Boss 或 存活指定时间。
- **失败**：玩家生命值归零。
