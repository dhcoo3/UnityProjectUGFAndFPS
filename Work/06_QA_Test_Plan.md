# 6. 质量保证与测试用例文档 (QA & Test Plan)

## 6.1 测试重点
- **逻辑确定性**：验证在不同帧率下，定点数位移是否完全一致。
- **状态切换鲁棒性**：连续快速按键、在受击瞬间切换状态是否会导致逻辑卡死。
- **配置表健壮性**：表格填写错误（空值、非法 ID）时，系统是否能优雅报错而不崩溃。

## 6.2 冒烟测试用例 (Smoke Tests)
1. **启动测试**：从初始化到主界面，无报错日志。
2. **关卡进入**：点击关卡 -> 加载 -> 玩家出生成功。
3. **基础战斗**：
    - 移动、跳跃、射击功能正常。
    - 杀死怪物后产生掉落。
4. **存盘/读盘**：战斗结束后数据正确保存。

## 6.3 性能压力测试
- **同屏单位数**：50个动态单位时，逻辑帧耗时是否在 10ms 以内。
- **内存泄漏**：反复进出战斗场景 20 次，观察内存增长曲线。

## 6.4 验收流程
- 开发人员自测 (Unit Test)。
- 提交合并请求前进行冒烟测试。
- 每周五进行一个稳定版本的关卡全流程跑通。
