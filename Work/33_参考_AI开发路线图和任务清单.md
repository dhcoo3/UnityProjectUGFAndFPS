# AI游戏开发进度文档

## 文档目标

本文档为 AI 协助游戏开发提供详细的执行路径，每个章节均包含完整的任务分解、MCP 工具使用、配置表操作和验证检查清单。

**核心原则**：先闭环，后扩展。优先完成单角色、单地图、单关卡、单结算的可玩闭环，再迭代扩展。

### 1.3 地图生成参数与矩阵定义

#### **地图符号标准**

用矩阵表示地图结构，符号含义如下：

| 符号 | 含义 | 碰撞特性 | 备注 |
| :--- | :--- | :--- | :--- |
| `0` | 空气（无碰撞） | 无 | 玩家和怪物可通过 |
| `1` | 地面（固定碰撞） | 有 | 玩家/怪物站立 |
| `2` | 单向平台 | 单向 | 可从下方通过，上方可站立 |
| `3` | 移动平台 | 有 | 会移动，需在 PlatformDef 配置终点 |
| `P` | 玩家出生点 | 有 | 玩家初始位置（该位置也是地面） |
| `M` | 怪物刷新点 | 无 | 标记怪物触发位置，不占用地面 |
| `E` | 关卡结束点 | 无 | 标记胜利触发区域 |

**矩阵方向**：
- 行（纵轴）：从上到下为 Y 轴递减（上面是较高的 Y 值）
- 列（横轴）：从左到右为 X 轴递增
- 每个单元格代表一个 Tile（1 Tile = 1 游戏单位）

#### **第一步：MCP 创建地图**

**目标**：使用 MCP 的 Tilemap 工具根据上述矩阵生成地图

**具体操作**：

1. 启动 Unity Editor（MCP 会自动连接）

2. 使用 MCP 的 Tilemap 创建工具
   - 创建新场景：`ChapterO1_MapO1`
   - 添加 `Tilemap` 层级：
     - `Collision`（逻辑阻挡层）：按矩阵中 `1` 的位置放置 Tile
     - `Platform`（平台层）：按矩阵中 `2` 的位置放置单向 Tile
     - `Visual_Back`（背景）：添加视觉装饰
     - `Visual_Mid`（主地表）：地面贴图
     - `Visual_Front`（前景）：前景遮挡与装饰

3. 根据矩阵放置地形：
   - 按矩阵第 Y=1 行放置主路径地面（30 个地面 Tile）
   - 按矩阵第 Y=2-3 行放置平台结构
   - 在 `P` 位置标记玩家出生点
   - 在 `M` 位置标记怪物刷新点（不需要放置 Tile，仅用于配置参考）
   - 在 `E` 位置标记关卡结束点

4. 调整视觉：
   - 添加背景和装饰 Tile（不参与碰撞）
   - 确保高度视觉清晰

**验证点**：
- [ ] 场景可在 Unity Editor 中正常加载
- [ ] Collision 层包含所有地面 Tile
- [ ] 玩家出生点在 (0, 1)
- [ ] 关卡结束点在 (29, 1)

## 5. 常见问题与排查清单

### Q：地图碰撞导出后无法加载

**排查**：
- [ ] 导出路径是否正确？应为 `Assets/HotAssets/Scene/SceneAsset/`
- [ ] 文件名是否与 MapData.Name 一致？
- [ ] 碰撞数据是否为空？检查 Excel 中的字段

### Q：怪物刷不出来

**排查**：
- [ ] MonsterSpawner 的 MonsterId 是否有效？
- [ ] TriggerX 条件是否满足？在 LogicUpdate 中添加日志验证
- [ ] MaxCount 是否 > 0？
- [ ] MonsterSpawner.Initialize 是否被调用？

### Q：技能无法释放

**排查**：
- [ ] SkillDef 是否存在？
- [ ] TimelineId 是否有效？
- [ ] Cost 资源（ammo / stamina）是否充足？
- [ ] CoolDown 是否已重置？

### Q：AI 怪物卡住不动

**排查**：
- [ ] AIGroupDef.ActionIdList 是否为空？
- [ ] AIActionDef 的 Conditions 是否过严格？
- [ ] 怪物是否被 Buff 冻结？
- [ ] 是否陷入无限地形碰撞检测？

### Q：结算流程卡住

**排查**：
- [ ] FightProxy.EndFight 是否正确实现？
- [ ] GameOverProcedure.OnEnter 是否被调用？
- [ ] 结算 UI 是否正确注册？

## 7. 备注与注意事项

- **定点数使用**：所有战斗数值必须使用 `fix`、`fix2`、`fix3`，禁止使用 `float`
- **日志规范**：使用 `UnityGameFramework.Runtime.Log` 而不是 `Debug.Log`
- **字符串处理**：高频字符串处理使用 `ZString`
- **NULL 检查**：对外部对象调用需要做 NULL 检查和容错处理
- **配置驱动**：尽量不要写死魔数，所有可配置内容都放到对应的 Const 类或配置表
