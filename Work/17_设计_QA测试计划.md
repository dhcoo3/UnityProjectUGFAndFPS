# 6. 质量保证与测试用例文档 (QA & Test Plan)

## 6.1 文档目标

本文档用于定义项目测试范围、测试分层、关键风险和发布准入标准，确保以下工作可执行：

- 开发自测
- 配置联调测试
- 主流程回归
- 战斗专项测试
- 性能与稳定性验证

本项目测试重点不是“测得多”，而是优先覆盖当前工程最容易出问题的链路：

- 流程切换
- 定点逻辑
- 表驱动配置
- 战斗状态切换
- 重复进入和资源回收

## 6.2 质量目标

首个可玩版本的质量目标如下：

- 主流程可从启动稳定进入关卡并完成结算
- 战斗逻辑在不同机器和不同帧率下不出现明显结果漂移
- 配置错误有明确日志，不能直接把游戏打崩
- 重复进出关卡和切换界面不出现大规模状态残留
- 高压场景下帧率与内存表现可接受

## 6.3 测试分层

### 6.3.1 开发自测

适用范围：

- 单个功能开发完成后
- 配置表改动后
- 资源接入后

最低要求：

- 能编译
- 功能主路径可跑通
- 无持续刷屏报错
- 影响范围内主流程已回归

### 6.3.2 冒烟测试

适用范围：

- 每次合并前
- 每日构建后
- 提交测试版本前

目标：

- 快速确认版本没有严重阻塞问题

### 6.3.3 系统回归测试

适用范围：

- 周版本
- 里程碑版本
- 大范围配置或流程调整后

目标：

- 验证已有功能未被新改动破坏

### 6.3.4 专项测试

专项重点包括：

- 定点同步
- 战斗状态切换
- AI 行为稳定性
- 配置表容错
- 性能与内存

## 6.4 关键风险清单

当前项目的高风险点如下：

- `Procedure` 切换与场景加载事件时序
- `UIModuleManager` 模块注册和重复打开
- `GameProxyManger` 初始化与释放顺序
- `FightProxy / MainDungeonProxy` 进入战斗链路
- `Timeline / Skill / Bullet / Damage` 的事件时序错位
- `MonsterSpawner / AI` 在边界地形中的卡死与抖动
- 反复进出关卡后的对象池残留和资源泄漏

## 6.5 冒烟测试清单

### 6.5.1 启动与流程

1. 启动游戏，进入预加载页，无阻塞报错
2. 预加载完成后正常进入登录场景
3. 登录界面可打开，按钮可点击
4. 进入章节界面，不出现空引用和缺资源

### 6.5.2 关卡进入

1. 选择关卡并点击开始
2. 进入加载状态
3. 加载完成后地图出现
4. 玩家角色出生成功
5. HUD 正常显示

### 6.5.3 基础战斗

1. 玩家可移动、跳跃、切换朝向
2. 普攻可释放，子弹生成正常
3. 子弹命中怪物后怪物掉血
4. 怪物死亡后能正确销毁或回收
5. 伤害飘字和命中特效正常出现

### 6.5.4 结算退出

1. 完成关卡后正常进入结算
2. 退出到章节界面或主界面
3. 再次进入关卡仍然正常

## 6.6 战斗专项测试

### 6.6.1 定点逻辑

重点验证：

- 不同帧率下移动距离是否一致
- 不同帧率下子弹命中时机是否一致
- 击退、跳跃、坠落是否出现明显漂移

通过标准：

- 核心战斗结果不因帧率变化而产生可感知差异

### 6.6.2 状态切换鲁棒性

重点场景：

- 连续快速输入移动和跳跃
- 受击瞬间触发技能或冲刺
- 死亡前一帧释放技能
- 换弹中切场景或结束战斗

通过标准：

- 不出现卡死、无限硬直、状态机乱跳和输入锁死

### 6.6.3 技能与时间线

检查点：

- 技能释放后时间线节点按顺序执行
- 动画、子弹、AOE、Buff 触发时机正确
- 技能被打断后残留状态被清理
- 多次连续释放不会重复挂载旧事件

### 6.6.4 AI 与刷怪

检查点：

- 怪物能按触发条件刷新
- AI 巡逻、索敌、追击、攻击可正常切换
- 怪物在平台边缘、墙角、封锁区不出现卡死
- 波次完成条件正确，不早结算、不漏结算

## 6.7 配置表专项测试

需要重点验证以下异常输入：

- 空字段
- 非法 ID
- 引用不存在的外键
- 负数值或越界值
- 重复 ID

通过标准：

- 配置错误有清晰日志
- 游戏可安全阻断或降级，不直接崩溃
- 错误定位能明确到表名、字段名、记录 ID

## 6.8 UI 专项测试

至少覆盖：

- 界面打开、关闭、重复打开
- 按钮点击去重
- 章节入口状态刷新
- HUD 信息与战斗状态同步
- 结算面板在胜利和失败两种情况下表现正确

重点关注：

- 残留旧数据
- 多次进入后重复订阅事件
- 切场景后 HUD 未释放

## 6.9 性能与稳定性测试

### 6.9.1 压力场景

建议构造以下压力测试：

- 同屏 `50` 个动态单位
- 高频子弹、AOE、伤害飘字同时触发
- Boss 战叠加多个大特效
- 连续进出战斗场景 `20` 次

### 6.9.2 观察指标

- 逻辑帧耗时
- 渲染帧耗时
- GC 频率
- 内存曲线
- 场景切换后对象数量是否回落

建议首版目标：

- 50 个动态单位时逻辑帧耗时尽量控制在 `10ms` 以内
- 连续进出场景 20 次后内存曲线无持续异常上升

## 6.10 缺陷等级定义

- `S1 阻塞`：无法进入游戏、无法进入关卡、必现崩溃、无法结算
- `S2 严重`：核心功能异常、主流程可继续但体验严重损坏
- `S3 一般`：局部功能错误、有规避方法
- `S4 轻微`：表现问题、文案问题、低风险边缘问题

处理原则：

- `S1` 和 `S2` 不允许带入提测版本
- `S3` 需评估影响面和修复时机
- `S4` 可进入版本已知问题列表

## 6.11 测试节奏建议

- 开发完成后立即自测
- 每日构建做一次冒烟
- 每周至少做一次主流程回归
- 每个里程碑前做一次专项性能与稳定性检查

## 6.12 发布准入标准

版本可进入发布候选的最低条件：

- 启动到关卡结算主流程全部通过
- 关键配置表无已知阻塞错误
- 主战斗链路无 `S1 / S2`
- 连续进出关卡和重复开关界面稳定
- 已知问题列表和规避说明完整

## 6.13 测试记录模板

每次测试建议至少记录：

- 测试版本号
- 测试时间
- 测试人
- 测试场景
- 实际结果
- 结论
- 缺陷编号
- 是否回归通过

## 6.14 配置表级测试映射

为了避免“功能能进，但配置一改就坏”，建议测试按表做最小覆盖。

### 6.14.1 实体表

`Hero`

- 角色资源缺失时是否有日志
- `MoveSpeed` 极值是否导致穿模
- `BodyRadius / HitRadius` 极值是否破坏碰撞
- 墙跳参数异常时是否还能恢复状态

`Monster`

- `AI` 空值或非法值是否能报错
- `SkillList` 空列表时怪物是否仍可正常移动
- `SearchRange` 极值是否出现全图追击
- `Asset / AnimGroup` 错误时是否能准确定位

### 6.14.2 关卡表

`MainChapter`

- 非法 `MapId` 是否阻断并报错

`MapData`

- `Name` 不存在时是否准确报错
- `Birth` 在阻挡里时是否能被发现
- `PlatformDataId` 配错时是否只影响平台而不是整张图崩掉

`PlatformDef`

- 起终点异常时平台移动是否异常
- 半宽半高配置错误时角色站立判定是否异常

`MonsterSpawner`

- `TriggerX` 过小是否开局立即刷怪
- `SpawnX / SpawnY` 在阻挡中是否卡死
- `MaxCount` 极值时是否造成性能突刺

### 6.14.3 战斗表

`SkillDef`

- 消耗和条件冲突时是否无法释放
- `TimeLine` 非法时是否准确报错

`TimeLineDef`

- 节点顺序混乱时是否仍可执行
- `Duration` 小于最后节点时间时是否异常

`SkillBulletDef / BulletDef`

- 子弹速度、时长、碰撞半径极值回归
- `HitAlly / HitFoe` 组合是否符合预期
- `RemoveOnObstacle` 与地图碰撞是否一致

`SkillAoeDef / AoeDef`

- AOE 半径、角度、持续时间是否与视觉一致
- Tick 事件频率过高时是否出现异常

`BuffDef`

- `Permanent`、`Duration`、`MaxStack` 组合是否正确
- `OnOccur / OnRemoved / OnTick` 是否重复触发
- 属性增减和控制状态是否正确恢复

### 6.14.4 UI 表

`GameModule`

- 模块 `Entry` 路径错误时是否有明确报错
- 模块重复打开时是否被拦截

## 6.15 按章节的回归优先级

### 6.15.1 第一章回归重点

- 进图
- 出生
- 移动跳跃
- 普攻射击
- 基础刷怪

### 6.15.2 第二章回归重点

- 高低差地图
- 动态平台
- 远程怪索敌
- 技能和 AOE

### 6.15.3 第三章回归重点

- Boss 技能循环
- 多阶段战斗
- 结算和退出
- 重开和再次进入
